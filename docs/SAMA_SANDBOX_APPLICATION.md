## القسم ب-1: تعريف المنتج

### المنتج الرئيسي: منصة تقسيط مشتريات الوقود

دربي هي منصة تقنية مالية متخصصة تقدم خدمة **"الشراء الآن والدفع لاحقاً" (BNPL)** حصرياً لقطاع الوقود في المملكة العربية السعودية. تعمل المنصة كوسيط مالي بين:

1. **المستهلكين (الأفراد والشركات):** الذين يحتاجون لتقسيط مشتريات الوقود
2. **محطات الوقود (التجار):** الذين يرغبون في زيادة مبيعاتهم عبر تسهيلات الدفع
3. **دربي (المنصة):** التي تدير عملية التقسيط والتسوية وتحصل على عمولة

### الفجوة في السوق السعودي

| التحدي | الوضع الحالي | حل دربي |
|--------|-------------|---------|
| تكلفة الوقود المرتفعة | لا توجد خيارات تقسيط للوقود | تقسيط مشتريات الوقود على 3-6 أشهر |
| إدارة ميزانية الوقود | صعوبة التتبع والتخطيط | تتبع ذكي مع تنبيهات استهلاك |
| أسطول الشركات | دفع فوري لتكاليف الوقود | تقسيط مرن لأساطيل الشركات |
| تحسين الاستهلاك | لا توجد أدوات ذكية | محرك سنافي للقرار الذكي |

### نموذج العمل

```
العميل ← طلب تقسيط وقود ← دربي (تحقق + موافقة) ← محطة الوقود
                                    ↓
                        تسوية فورية للمحطة (97%)
                        عمولة دربي (3%)
                                    ↓
                        أقساط شهرية من العميل
```

**مسار الأموال (Money Flow):**
1. **الخطوة 1:** العميل يطلب تعبئة وقود عبر المنصة
2. **الخطوة 2:** دربي تتحقق من الأهلية والحدود الائتمانية
3. **الخطوة 3:** يتم خصم المبلغ من حد العميل الائتماني
4. **الخطوة 4:** دربي تدفع للمحطة فوراً (ناقص العمولة 3%)
5. **الخطوة 5:** العميل يسدد الأقساط الشهرية لدربي

## القسم ب-2: القيمة المضافة

### تحسين إدارة التدفقات النقدية للمستهلكين

1. **تحويل النفقات الكبيرة إلى أقساط مُدارة:**
   - بدلاً من دفع 500 ريال دفعة واحدة، يمكن للعميل تقسيطها على 3-6 أشهر
   - يساعد في تنظيم الميزانية الشهرية وتجنب الضغوط المالية المفاجئة

2. **الشفافية المالية الكاملة:**
   - عرض تفصيلي لكل عملية: المبلغ، الأقساط، تواريخ الاستحقاق
   - تنبيهات استباقية قبل مواعيد السداد
   - تقارير استهلاك الوقود الشهرية

3. **محرك سنافي (Snafi AI Engine):**
   - نظام ذكاء اصطناعي يحلل أنماط استهلاك الوقود
   - يقدم توصيات ذكية بأفضل وقت ومكان للتعبئة
   - يتنبأ بمستوى خزان الوقود ويُنبه قبل نفاده
   - يقلل التكلفة الإجمالية للوقود عبر التعبئة المثلى

4. **تمكين أصحاب الأعمال الصغيرة:**
   - سائقو التوصيل (هنقرستيشن، مرسول، جاهز) يمكنهم تقسيط الوقود
   - أصحاب الشاحنات والنقل يستفيدون من التدفق النقدي المحسّن
   - الشركات الصغيرة تدير أساطيلها بكفاءة أعلى

## القسم ج-2: نطاق الاختبار (Sandbox Scope)

### المعايير الكمية لفترة البيئة التجريبية

| المعيار | القيمة | المبرر |
|---------|--------|--------|
| **عدد المستخدمين** | 500 مستخدم | كافٍ لاختبار الأنماط السلوكية والمخاطر |
| **عدد المحطات** | 20 محطة | تغطية 3-5 مدن رئيسية |
| **حد العملية الواحدة** | 500 ريال | يغطي تعبئة خزان كامل |
| **الحد اليومي للمستخدم** | 1,000 ريال | يمنع الإفراط في الاستخدام |
| **الحد الشهري للمستخدم** | 5,000 ريال | إدارة محكمة للمخاطر |
| **مدة الاختبار** | 12 شهراً | كافية لجمع بيانات موسمية |
| **المدن المستهدفة** | الرياض، جدة، الدمام | أكبر 3 أسواق في المملكة |

### إجراءات إدارة المخاطر في البيئة التجريبية

1. **فصل البيانات (Data Segregation):**
   - كل عملية تُعلَّم بـ `environment: "sandbox"` في قاعدة البيانات
   - فصل تام بين بيانات الاختبار والإنتاج
   - لوحة تحكم مخصصة لرقابة ساما (`/admin/monitoring`)

2. **حدود العمليات المتعددة المستويات:**
   - حد العملية الواحدة (max_single_transaction)
   - حد يومي (daily_limit)
   - حد أسبوعي (weekly_limit)  
   - حد شهري (monthly_limit)

3. **المراقبة اللحظية (Real-time Monitoring):**
   - تتبع كل عملية مالية لحظياً
   - تسجيل مسار الأموال الكامل (customer → darby_escrow → station)
   - إشعارات فورية عند تجاوز الحدود
   - تصنيف مخاطر تلقائي (risk scoring)

4. **سجلات تدقيق شاملة (Audit Trail):**
   - سجل `money_flow_logs`: يوثق كل حركة مالية
   - سجل `limit_breaches`: يوثق كل محاولة تجاوز حد
   - سجل `audit_logs`: يوثق كل إجراء إداري
   - البيانات محفوظة بشكل دائم وغير قابلة للتعديل (immutable)

## القسم د: الابتكار

### ما يميز دربي عن منصات BNPL التقليدية

**1. التخصص في قطاع الوقود:**
- دربي ليست منصة BNPL عامة، بل متخصصة حصرياً في الوقود
- هذا التخصص يتيح تقييم مخاطر أدق وفهم أعمق للسوق
- نماذج التسعير مصممة خصيصاً لنمط استهلاك الوقود

**2. محرك سنافي (Snafi AI Engine) - الابتكار الأساسي:**

محرك سنافي ليس مجرد نظام تقسيط، بل مساعد ذكي متكامل:

| الميزة | الوصف | الفائدة |
|--------|------|---------|
| **قياس مستوى الخزان** | تتبع ذكي لمستوى الوقود | معرفة الوقت الأمثل للتعبئة |
| **تحليل الاستهلاك** | نماذج ML لتحليل أنماط القيادة | تقليل التكلفة بنسبة 10-15% |
| **توصيات ذكية** | اقتراح أفضل محطة وكمية | توفير الوقت والمال |
| **تعلم مستمر** | تحسين الدقة مع كل استخدام | نسبة دقة تتجاوز 85% |
| **تنبيهات استباقية** | إشعار قبل نفاد الوقود | تجنب المواقف الطارئة |

**3. أتمتة العملية بالكامل:**

```
الطريقة التقليدية:
العميل → يذهب للمحطة → يدفع نقداً → لا سجل إلكتروني

طريقة دربي:
العميل → يفتح التطبيق → يختار المحطة → موافقة فورية → QR Code
   ↓
المحطة تمسح الكود → تعبئة → تسوية تلقائية → أقساط تُحسب
   ↓
سنافي يحلل → يتعلم → يحسّن التوصيات المستقبلية
```

**4. تقليل الخطأ البشري:**
- لا تدخل يدوي في حساب المبالغ أو الأقساط
- تحقق آلي من الأهلية عبر (Nafath + SIMAH + GOSI)
- تسوية تلقائية مع المحطات
- إدارة مخاطر مؤتمتة بالكامل

**5. نظام تحقق متعدد الطبقات (KYC/AML):**
- **نفاذ (Nafath):** التحقق من الهوية الوطنية
- **سمة (SIMAH):** تقرير ائتماني وتقييم الأهلية
- **التأمينات (GOSI):** التحقق من الحالة الوظيفية
- **مكافحة غسل الأموال (AML):** فحص القوائم السوداء

## القسم هـ: الأمان والحماية

### طبقات الحماية المُطبقة

1. **Rate Limiting:** 10 طلبات/دقيقة/IP
2. **Input Sanitization:** تنقية المدخلات من الأكواد الخبيثة
3. **ID Validation:** التحقق من صيغة المعرفات (UUID/Alphanumeric)
4. **Business Validation:** حدود عملية (1-200 لتر، أسعار مرجعية)
5. **Server-Side Pricing:** تسعير من الخادم فقط (لا يمكن التلاعب)
6. **Credit Limit Checks:** التحقق من الحد الائتماني قبل كل عملية
7. **ORM Protection:** استعلامات محصنة ضد SQL Injection
8. **Sandbox Isolation:** فصل تام بين بيئة الاختبار والإنتاج

### التهديدات المعالجة

| التهديد | الحماية |
|---------|---------|
| SQL Injection | ORM + Parameterized Queries |
| XSS | Input Sanitization + CSP |
| Brute Force | Rate Limiting + Account Lockout |
| Parameter Tampering | Server-side Validation |
| Price Manipulation | Server-side Pricing Only |
| Unauthorized Access | RBAC (27 دور، 14 قسم) |

## الخلاصة

دربي تقدم حلاً مبتكراً ومتخصصاً لقطاع الوقود في المملكة العربية السعودية، مع:
- **ابتكار تقني حقيقي** عبر محرك سنافي للذكاء الاصطناعي
- **إدارة مخاطر محكمة** عبر حدود متعددة المستويات وبيئة تجريبية معزولة
- **شفافية كاملة** عبر سجلات تدفق الأموال ولوحة رقابة لحظية
- **امتثال تنظيمي** متوافق مع متطلبات ساما ومعايير البيئة التجريبية
- **قيمة مضافة واضحة** للمستهلكين والتجار والاقتصاد الوطني
